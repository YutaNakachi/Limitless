## 🔮 ゲームコンセプト
- **馴染み深くキレのある操作性**
  - 移動、ジャンプ、ダッシュなどの基本挙動は、直感的かつ「きびきびとしたスピード感」を徹底的に再現。
- **呪術球（ボール）を蹴り出す独自の攻撃**
  - 自身の周囲を公転する球をタイミングよく蹴り飛ばして攻撃。
  - 様々な種類の呪術球が存在し、戦況を大きく変える特殊能力を発動するものも！
- **戦術に即興性を生む「ガチャ要素」**
  - リロードごとにランダム装填される呪術球（強力な球ほど低確率）に応じ、その場で最適な立ち回りを瞬時に組み立てる即興の戦術性。

## 🛠️ 特徴的なゲームシステム

### 1. 呪術球・オービットシステム（Orbit System）
プレイヤーの周囲を公転し続ける6つの呪術球（固定スロット管理）。
- **属性（一例）**: 「白色（通常弾）」「赤色（爆発）」「青色（吸引）」
- **目押し技術×運要素**: プレイヤーは**自身の正面（キックポイント）に回ってきた球だけ**を蹴り出すことが可能。回るタイミングを見極め、狙いの色を撃ち抜く「技術」と最適な球を引き寄せる「運」を掛け合わせた唯一無二のシステム。

### 2. 属性融合・共鳴システム（Resonance System）
オービット上に特定の属性球が同時に装填された際、それらを掛け合わせて超強力な上位能力へと昇華させる「役（コンボ）」システム。

「赤色（爆発）」と「青色（吸引）」の2つが同時にスロットに存在する場合、プレイヤーの任意入力によってこの2つが融合し、最上位の紫色の球**【「紫（虚式・茈）」】**へと変貌する。

## ⌨️ 基本操作仕様

### ゲームパッド
- **左右移動**: `左スティック` を左右に入力
- **しゃがみ**: `左スティック` を下に入力
- **ジャンプ**: `A` ボタン（空中ジャンプを含め、最大2段まで可能）
- **ダッシュ**: `左スティック` を左右に**すばやく2回入力** または `B` ボタン
- **キック**: `X` ボタン（正面のキックポイントに来た球を射出）
- **エイム**: `左スティック` （左スティックを倒した方向に球を射出）
- **スマッシュキック**: `左スティック` をはじき入力後、すばやく `X` ボタン（威力の高い球を射出）
- **リロード**: `Y` ボタン（空きスロットへランダム装填）
- **レゾナンス(融合)システム発動**: `RB` ボタン（スロットに存在するボールの組み合わせに応じて発動）

### キーボード・マウス
- **移動**: `A` / `D` キー（左右移動）
- **しゃがみ**: `S` キー
- **ジャンプ**: `W` キー または `Space` キー（空中ジャンプを含め、最大2段まで可能）
- **ダッシュ**: `A` または `D` キーを**すばやく2回入力** または `Left Shift` キー
- **キック**: `マウス左クリック` （正面のキックポイントに来た球を射出） または `Enter` キー
- **リロード**: `R` キー（空きスロットへランダム装填）
- **レゾナンス(融合)システム発動**: `M` キー（スロットに存在するボールの組み合わせに応じて発動）

### 3. BallAbility（オブジェクト指向による属性拡張）
球の特殊効果は基底クラス `BallAbility.cs` を継承し、Manager側を汚さずに新属性を追加できる設計（ポリモーフィズム）を徹底します。
1. **無色（通常弾）**: 特殊効果なし。真っ直ぐ飛んでいき、敵に直接ダメージを与える。
2. **赤色（爆発・赫）**: 着弾時（`OnHit`）にその場で円形の範囲爆発（OverlapCircle等）を起こし、周囲の敵やブロックを巻き込む。
3. **青色（吸引・蒼）**: 着弾時に空間歪曲のトリガーを生成。周囲の敵オブジェクトの Rigidbody2D を着弾中心点に向けて持続的に引き寄せる（擬似ブラックホール）。
4. **👑 紫色（融合・茈）**: 【赤】と【青】がスロットに揃った時のみ生成される最上位クラス。コライダーの `isTrigger` を利用して敵や一部のブロックを「無条件で貫通（破壊）」し、直線上のすべてを消滅させる絶対判定の挙動を持つ。

- Ballの飛び方の特徴は、各BallPrefabにアタッチされたRigidbody2DのParameterで調整する（**Linear Dumpling**, **Gravity Scale**, **Mass**）
- Ballが蹴られてからDestroyされるまでの時間は、各BallPrefabにアタッチされたBallControllerの**BallLifeTime**(s)で調整する。
- BallPrefabにアタッチされたRigidBody2Dの'CollisionDetection'は**Continuous**にすること。これにより壁のすり抜けを防げる。

### 4. Map / ステージ の構成
無限生成に対応するため、地形やギミックはすべてコンポーネント単位でモジュール化します。
- **Grid / Tilemap**: ステージの基本地形（足場）。効率的な描画のため `Tilemap Collider 2D` および `Composite Collider 2D` を適用。
- **OneWayPlatform**: 空中に配置される足場ギミック。プレイヤーが下からすり抜けて乗れるワンウェイプラットフォーム仕様などを検討。

* **スライディングバグ・二重動作の防止（厳格なインターロック）**
  ノックバックやスタン、あるいは攻撃終了時のクールダウンによって `MobStatus` の状態が強制的に書き換わった際、AI側（`BatAI`）や攻撃側（`MobAttack`, `PlayerShoot`）の `FixedUpdate` がそれを自動検知し、**内部の全アクションフラグ（ため・突進・攻撃など）を強制リセットしてNormalに引き戻す回路**が組まれています。これにより、キャラクターが硬直を無視して滑りながら攻撃し続けたりするバグをシステムレベルで根絶しています。

* **ポリモーフィズムによる共通化と高い量産性**
  HP管理、ダメージ処理、初期無敵時間、状態定義といった「生命体としての共通ロジック」はすべて親クラス `MobStatus` に集約。それを継承した `PlayerStatus` と `EnemyStatus` は、**死亡時や攻撃時の固有の分岐処理（ゲームオーバーを呼ぶか、3秒後に消えるか等）を書くだけで済む**ため、コードの重複が一切ありません。今後、新しいエネミーを追加する際も、AIロジックのみを新規実装するだけでHP追従や死亡消滅などの全システムが最初から自動で適用されます。

* **手触りの良さ（Juice）とデバッグ性の両立**
  * **入力バッファ**: `PlayerShoot` には0.1秒の先行入力猶予があり、レバーを離した直後のキックでも意図した方向へ正確にシュートが放てます。
  * **可視化Gizmo**: ニュートラル（緑）と入力中（赤）で色が変わり、クランプ限界線が黄色でシースルー表示されるため、エディタ上で弾道予測が一目でデバッグ可能です。

## 🎬 演出制御システム（FxManager & FxPresetData）

ゲーム内の打撃感（プレイフィール）を最大化するため、時間制御（ヒットストップ）、空間振動（カメラシェイク）、触覚フィードバック（ゲームパッドハプティクス）、および被弾オブジェクトの微振動をミリ秒単位で一元管理する総合演出システム。

---

### 1. ⚙️ アーキテクチャ概要

本システムは、UniTaskによる非同期処理を全面採用し、異なる演出アセットを単一のラベル文字列（例: `"Resonance"`, `"Damaged"`）だけで安全に呼び出せる設計になっています。

* **`FxPresetData`（ScriptableObject）**
  演出のパラメータ（停止時間、スロー倍率、カメラの揺れ幅、コントローラーの左右モーター振動強度、間引き設定など）をアセット側に完全に分離。ゲームデザイナーや開発者がコードを触らずに、打撃感のインスペクター調整を完結できます。
* **`FxManager`（MonoBehaviour / Singleton）**
  `Time.timeScale` を弄る「画面全体の時間軸ハック」と、「コントローラー振動」「カメラ位置」の同期制御を行うコアエンジン。

---

### 2. 🛡️ 核心機能とこだわり仕様

#### 🔹 ① 大量撃破時の演出間引き（グローバル・クールタイム回路）
範囲攻撃や広範囲の融合（レゾナンス）によって敵を同時に大量撃破した際、すべてのヒットストップ演出がバカ正直に重なると、ゲームが完全にフリーズしたような不快なプチフリーズ（暴発）を引き起こします。
* **最適化ロジック**: 
  アセット側の `useCoolTime` が有効な演出が短時間（デフォルト: `0.08秒` 内）に連続で呼ばれた場合、システムは「画面全体の一時停止（ヒットストップ）やカメラシェイク」を自動的にミュート（間引き）します。
* **手応えの維持**: 
  全体演出は間引きつつも、殴られた敵個別の「被弾オブジェクトの微振動」だけは独立して100%実行されるため、**「ゲームのテンポを一切崩さない爽快感」と「個々の敵をしっかり殴った確かな手応え」を完璧に両立**させています。

#### 🔹 ② コントローラー振動（ハプティクス）の異周波同期
New Input System の API をハックし、演出データに基づいた左右のモーター個別制御を行っています。
* **左モーター（低周波）**: 「ドンッ」という重い一撃や爆発などの物理的な衝撃を表現。
* **右モーター（高周波）**: 「チリチリ」「カリッ」とした斬撃やエネルギー結合などの繊細な手触りを表現。
* これらをヒットストップの時間軸（タイムスケール）と完全に同期させて駆動させています。

#### 🔹 ③ 連続被弾時の「位置ズレ・振動暴発」防止回路
カメラシェイクやオブジェクトの微振動、ゲームパッドの振動を非同期（Async）で回す際、演出中にさらに強い一撃が重なった場合の割り込み・上書き制御を徹底しています。
* **時間の上書き**: 
  新しく要求された停止時間が残りの停止時間よりも長い場合のみ、古い `CancellationTokenSource (CTS)` を安全にキャンセルし、最新の強力な演出へと途切れることなく滑らかに上書き遷移（インターラプト）させます。
* **絶対座標の完全復帰**: 
  カメラやオブジェクトの原点位置（`originalCameraPos` 等）を `Start` 時に厳密にキャッシュ。どれだけシェイク中にキャンセルや割り込みが入っても、`finally` ブロックが100%駆動して強制的にカメラやオブジェクトを元の座標へピタッと戻すため、**「連続被弾するとカメラの位置が徐々に画面外にズレていく」というアクションゲーム特有のバグを根本からシャットアウト**しています。

## 🔊 オーディオシステム（SoundManager & SoundDataAsset）

ゲーム内のすべてのBGMおよびSE（効果音）を統合管理するオーディオシステム。ScriptableObjectによるデータの一元化に加え、独自の秒単位トリミングおよび高度なループ制御機能を搭載しています。

---

### 1. ⚙️ アーキテクチャ概要

本システムは、オーディオデータのパラメータを保持するアセット型スクリプトと、それをシーンを跨いで再生・制御するマネージャーの2レイヤーで構成されています。

* **`SoundDataAsset`（ScriptableObject）**
  すべての音源アセット（AudioClip）とその設定（音量・ループ・再生範囲）を管理するマスターデータベース。インスペクター上で一元管理できるため、コードを書き換えることなく音量バランスやトリミング範囲の微調整が可能です。
* **`SoundManager`（MonoBehaviour / Singleton）**
  `DontDestroyOnLoad` によりシーン遷移後も破壊されずに常駐する、ゲーム内唯一の音響スピーカー。2D（UI・全画面）および3D（特定座標・空間音響）の双方に対応しています。

---

### 2. ⏱️ 核心機能：手動トリミング＆イントロ付きループ再生

Unity標準の `AudioSource.loop = true` では実現できない、**「任意の区間だけを切り取ってループさせる機能」** をコルーチンベースで独自に実装しています。

#### 🔹 BGM（背景音楽）の仕様
BGM再生時、Unity標準のループ機能は明示的にOFFにされ、内部の `BgmLoopRoutine` が毎フレーム再生時間を監視します。
* **イントロ付きループの実現**:
  アセット側に `startTime` と `endTime` を設定することで、「イントロ部分（$0 \sim \text{startTime}$）を初回のみ再生し、曲の終端（`endTime`）に達した瞬間、自動的に `startTime`（サビやメインループの頭）へジャンプしてループする」という、本格的なゲーム用BGM制御を完全自動で行います。

#### 🔹 SE（効果音）の仕様
SEアセット側でもループフラグ（`isLoop`）および再生区間の指定が可能です。
* **2D / 3Dサウンドの動的生成**:
  `PlaySE` または `PlaySEAtPosition` が呼ばれると、内部で一時的なGameObject（スピーカー）を動的に生成して再生を開始。単発SEの場合は、トリミング再生が終了した瞬間にGameObjectが自動的に破棄（デストロイ）され、メモリをクリーンに保ちます。
* **ループSEの一元管理と多重発動防止**:
  壁すり（`WallSliding`）の持続音など、ループ再生が必要なSEは内部の専用辞書（`_activeLoopSEs`）に登録されます。これにより、同じ音が重複して暴発するのを完璧に防ぎ、不要になったタイミングで `StopLoopSE("キー名")` を呼び出すことで、安全に再生を終了しオブジェクトを破棄します。

## 🌀 システム仕様：オービット＆レゾナンス（融合）システム

ゲームの核心となる、プレイヤーの周囲を回転するボールの挙動（オービット）と、それらを掛け合わせて上位弾を生成する融合（レゾナンス）の統合管理仕様。

---

### 1. オービット（軌道周回）システム

プレイヤーの周囲に最大6個のボールを動的に配置し、一定の軌道上で回転・管理する基本システム。

* **動的等間隔配置**
  手元にあるボールの数に関わらず、最大スロット数（`maxBalls: 6`）に基づいて常に均等な角度（$360^\circ \div 5 = 72^\circ$ 間隔）をリアルタイムに計算し、美しい円形陣形を維持します。
* **物理的な失速・キック判定の分離**
  一度キックされて手元を離れた（`isKicked == true`）ボールは、自動的にオービット計算の対象外となり、物理挙動（Rigidbody2D）へとシームレスに移行します。
* **時間停止への適応**
  ポーズ画面や後述するレゾナンス演出による時間停止（`Time.timeScale = 0`）の際、ボールの回転もプレイヤーの感覚と同期して自然に停止するよう、`Time.deltaTime` ベースで回転角（`currentAngle`）を制御しています。
* **オートリロード機能**
  手元の周回ボールが完全にゼロになったことを検知すると、一定の猶予時間を経て、空いているスロットへ確率（`Rate`）に基づいたランダムなボールが自動的に順次装填（リロード）されます。
* **確率ベースの動的抽選システム（Rate）**
  リロード時に生成されるボールは、各ボールアセットに設定された個別確率（`Rate`）を元に、毎回完全に動的な **「重み付きランダム抽選（ルーレット選択アルゴリズム）」** によって決定されます。
  
  * **抽選の仕組み**:
    1. リスト内の全ボールの `Rate`（重み）の合計値（`rateTotal`）を算出。
    2. $0 \sim \text{rateTotal}$ の間でランダムな浮動小数点数（`randomValue`）を生成。
    3. 各ボールの `Rate` を順番に累積（足し算）していき、生成したランダム値が累積値を超えた瞬間のボールを当選アセットとして選出。
  
  これにより、単なる均等なランダム（1/N）ではなく、「基本のコモン球は出現率を高く（例：Rate=70）」「強力なレア球や回復球は出現率を低く（例：Rate=5）」といった、ゲームの難易度やステージの緊張感を左右する細かなドロップバランスの調整を、インスペクター上の数値変更だけで完全制御できる設計になっています。

---

### 2. レゾナンス（融合）システム

手元にある2つの異なる属性ボールを消費し、レシピに基づいた強力な上位ボールを錬成する独自のクラフト・強化システム。

#### 🔹 融合レシピ（ResonanceRecipe）の決定ロジック
1. **素材の自動探索 (`TryFindMaterials`)**
   プレイヤーが融合ボタン（`RBボタン`）を押した瞬間、現在オービットしているボールの中からレシピに必要な「素材A」と「素材B」がスロット内に存在するかを、順不同・重複なしで高速探索します。
2. **優先度判定（Priority）**
   複数の融合レシピが同時に成立可能な状態である場合、レシピ側に設定された `priority`（数値が低いほど高優先）を比較し、最も優先度の高いレシピを自動選択します。
3. **同優先度のランダム選出**
   完全に同一の優先度を持つレシピが複数合致した場合は、内部ロジックによりランダムで1つが選出され、融合プロセスへ移行します。

---
